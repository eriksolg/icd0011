Projekti 4. osa

Lisage eelmises osas tehtud rakendusele funktsionaalsus, mis 
jätab meelde sisestatud tellimusi ja lubab neid pärast id järgi välja küsida.
Lisaks peaks tellimuse sisestamisel olema võimalik lisaks numbrile määrata 
ka tellimuse read. Tehtud töö märgistage tag-iga hw04.

Andmete hoidmiseks kasutage ServletContext-i atribuuti.

1. Eelmise osas ei olnud nõutud, et postitatud tellimus kuhugi salvestataks ka.
   Nüüd peaks olema võimalik seda tellimust hiljem vaadata. Millist tellimust 
   soovitakse vaadata on määratud id-ga.

   Eelmises osas tehtud kood tagastas tellimuse postitamise peale sisestatud 
   info ja lisas sellele id. Nüüd peaks sellesama id järgi olema hiljem 
   võimalus postitatud tellimust vaadata.

   Päringu näide:
   
   GET /api/orders?id=5 HTTP/1.1
   Host: localhost:8080

   Vastuse näide:
   
   HTTP/1.1 200 OK
   Content-Type: application/json
  
   { "id": 5, "orderNumber": "A123" }

2. Tellimus võib sisaldada ka infot tellimuse ridadest.

   Päringu näide:
   
   POST /api/orders HTTP/1.1
   Host: localhost:8080
   Content-Type: application/json

   { "orderNumber": "A456",
     "orderRows":[
       {"itemName":"CPU","quantity":2,"price":100},
       {"itemName":"Motherboard","quantity":3,"price":60}
     ]
   }

  Vastuse näide:
  
  HTTP/1.1 200 OK
  Content-Type: application/json

   { "id": 5,
     "orderNumber": "A456",
     "orderRows":[
       {"itemName":"CPU","quantity":2,"price":100},
       {"itemName":"Motherboard","quantity":3,"price":60}
     ]
   }

   Tellimuse hilisemal vaatamisel peaks need read ka näha olema.
   Json-i konverteerimiseks kasutage ObjectMapper-it või mõnda muud teeki.

3. See ülesanne ei ole tingimata kohustuslik. Kui see jääb tegemata saate 
   maksimaalse 5 punkti asemel 3.
   
   Lisage rakendusele võimalus, et tellimust saaks sisestada lisaks 
   Json-ile ka vormist postitatud andmete põhjal. Siin piisab ainult 
   tellimuse numbrist. Vastusena tagastab rakendus sisestatud tellimuse id.

   See võimalus asub aadressil (path) "orders/form" (teine Servlet).

   Postman-ist vastava päringu saatmiseks on "raw" Body asemel "x-www-form-urlencoded".

   Päringu näide:
   
   POST /orders/form HTTP/1.1
   Host: localhost:8080
   Content-Type: application/x-www-form-urlencoded
   Content-Length: 16

   orderNumber=A789

   Vastuse näide:
   
   HTTP/1.1 200 OK
   Content-Type: text/plain
   
   7
